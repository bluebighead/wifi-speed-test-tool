# ALIGNMENT.md - 需求对齐文档

## 1. 业务目标拆解

### 1.1 核心功能需求
| 功能模块 | 可量化指标 | 验收标准 |
|---------|-----------|---------|
| 网络速度检测 | 下载速度、上传速度、网络延迟 | 误差范围±10%，响应时间<2秒 |
| 信道占用分析 | 2.4GHz/5GHz双频段信号强度、占用率、干扰情况 | 实时更新频率≤1秒，数据准确率≥95% |
| 智能信道推荐 | 基于占用率推荐最优信道 | 推荐准确率≥90%，推荐响应时间<3秒 |
| 用户界面 | 操作便捷性、数据展示清晰度 | 用户操作步骤≤3步完成测速 |

### 1.2 非功能性需求
- **性能要求**：测速完成时间<30秒，信道分析实时更新
- **兼容性**：支持Windows 10/11主流系统
- **可靠性**：连续运行4小时无崩溃，异常处理覆盖率100%
- **易用性**：界面简洁，无需用户手册即可操作

## 2. AI能力边界

### 2.1 支持场景
✅ Windows平台WiFi信道扫描与分析
✅ 基于HTTP/HTTPS的网络速度测试
✅ 实时信道占用数据采集与可视化
✅ 智能信道推荐算法（基于占用率和干扰分析）

### 2.2 依赖数据
- Windows系统WiFi接口（通过netsh命令或Python库）
- 网络测速服务器（使用公共测速服务器）
- 系统网络接口信息

### 2.3 不支持场景
❌ Linux/macOS平台（需后续扩展）
❌ 有线网络测速（仅支持WiFi）
❌ 硬件级信号强度测量（依赖系统API）
❌ 离线模式（需要网络连接进行测速）

## 3. 技术选型

### 3.1 开发语言与框架
- **主语言**：Python 3.8+
- **GUI框架**：PyQt5（跨平台、功能丰富、文档完善）
- **数据处理**：NumPy、Pandas
- **图表展示**：Matplotlib
- **网络测速**：speedtest-cli / requests
- **WiFi扫描**：pywifi / subprocess调用netsh

### 3.2 依赖库清单
```
PyQt5>=5.15.0
pywifi>=0.1.0
matplotlib>=3.5.0
requests>=2.28.0
speedtest-cli>=2.1.0
numpy>=1.21.0
```

## 4. 风险识别

| 风险项 | 影响等级 | 应对措施 |
|-------|---------|---------|
| Windows权限限制 | 高 | 提示用户以管理员身份运行 |
| WiFi API兼容性 | 中 | 提供多种扫描方式备选方案 |
| 网络测速服务器不稳定 | 中 | 实现多服务器自动切换 |
| 实时性能瓶颈 | 低 | 优化数据采集频率，使用异步处理 |
